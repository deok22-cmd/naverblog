## 02 | 주제 선정 5대 기준 (우선순위 순)

1순위: 인물명 검색 — 실명 검색되는 인물 최우선. 제목 앞부분에 반드시 배치
2순위: 당일 이슈 — 사건·우승·발표·논란. 오전 발행이 골든타임
3순위: 진행형 이벤트 — 1R→2R→결과 시리즈로 연속 유입
4순위: 시즌 키워드 — 도시별·날짜별로 쪼개 경쟁 감소 (예: "도쿄 벚꽃 D-6")
5순위: 롱테일 스테디 — 레시피·제품 비교·정보성. 장기 트래픽 자산

⚠️ 주제 선정 후 반드시 팩트체크: 수치·날짜·인명·직함·결과값 전수 검증. 추측 절대 금지.

---

## 03 | 제목 — 키워드 3단 구조

빅키워드(앞 10자 이내) + 미들키워드 + 롱테일키워드 → 하나의 자연스러운 문장

3대 규칙:
- 빅키워드는 제목 앞 10자 안에 배치
- 키워드 나열 금지 → 자연스러운 문장으로
- 제목 = 검색자의 질문에 대한 정확한 답

나쁜 예 → 좋은 예:
- "김효주 LPGA 우승 파운더스컵 통산8승 넬리코다" → "김효주 파운더스컵 우승 — 16언더파, 넬리코다 1타 차 제압"
- "오사카 여행 맛집 추천 벚꽃 일정" → "오사카 벚꽃 여행 3박4일 — 도톤보리 맛집·나라 사슴공원 코스"
- "된장찌개 레시피 만드는법 재료" → "된장찌개 황금 레시피 — 멸치육수부터 끓이는 시간까지"

## 04 | HTML 원고 완성 구조

모든 HTML 파일은 아래 구조를 반드시 따른다. 배경색 #ffffff 고정.

CSS 필수 요소:
- body: max-width 800px, background #ffffff, font-family Noto Sans KR
- .category-badge: 카테고리 메인컬러A 배경의 뱃지
- h1: 제목 (font-size 1.65em)
- .post-meta: 날짜·카테고리 메타정보 (color #999)
- .lead: 리드문단 — border-left 4px solid 메인컬러A, background #f8f9fa
- h2: 소제목 — border-left 4px solid 메인컬러A (필수)
- h3: 서브소제목 — border-left 2px solid 메인컬러B
- .point-box: border 2px solid 메인컬러A, border-radius 10px
- .info-box: background 메인컬러A 투명도12%, border-radius 10px
- .tip-box: 여행 카테고리용 노란 팁 박스 (💡 TIP 접두)
- .recipe-step + .step-num: 레시피 카테고리용 번호 단계 박스
- .card-grid + .card: 스포츠 카테고리용 성적 카드 그리드
- table: th배경 메인컬러A, 짝수행 #fafafa
- .closing-box: 맺음말 — gradient 배경 (메인컬러A+메인컬러B 혼합)
- .hashtags: 해시태그 15개 내외, color 메인컬러A
- .source-notice: 출처 주의사항, font-size 0.78em, color #aaa (필수)
- .health-banner: 건강·의료 글 전용 경고 배너 (해당 시만)

HTML 본문 구성:
1. category-badge (카테고리 이모지 + 카테고리명)
2. h1 (제목)
3. post-meta (날짜 · 카테고리)
4. .lead (리드문단 — 구어체 2~3문장)
5. h2 소제목 4~6개로 구성된 본문
6. point-box 최소 1개
7. info-box 또는 tip-box 또는 card-grid (카테고리에 맞게)
8. table 1~2개 (테이블 남발 금지)
9. .closing-box (맺음말)
10. .hashtags (15개 내외)
11. .source-notice (출처 주의사항 — 필수)

## 05 | 카테고리별 레이아웃 스타일

스포츠 (야구/골프/축구/스포츠기타) — 매거진 에디토리얼형
- card-grid로 선수·팀 성적 카드 표시
- 경기 결과 테이블 포함
- 소제목: 단언형 위주 ("16언더파의 압도적 우승")
- 분량: 1,000~1,500자

여행 (일본/중국/해외/국내) — 일기체형
- 시간순 코스 서술 (오전→점심→오후→저녁)
- tip-box 2~3개 (입장료·교통·예약 팁)
- 소제목: 장소명 위주 ("도톤보리 — 오사카의 심장")
- 분량: 1,500~2,000자

음식레시피 — 일기체형
- 재료 목록 테이블 필수
- recipe-step 번호 박스로 단계 서술
- 소제목: 단계형 ("재료 손질", "육수 내기", "끓이기")
- 분량: 1,200~1,800자

연예/Kpop/영화/드라마/넷플릭스 — 에디토리얼형
- 스토리 흐름 중심 서술
- 소제목: 질문형 위주 ("왜 이 드라마가 화제인가?")
- 분량: 1,000~1,500자

## 06 | AI 판별 회피 기법 (모든 원고에 전부 적용)

1. 소제목 형태 혼재: 질문형 / 단언형 / 숫자형 섞기
2. 문단 길이 불규칙: 1줄 임팩트 문장 ↔ 4~5줄 상세 설명 번갈아
3. 도입부 구어체: 리드 문단 2~3문장 구어체로 시작
   예: "오늘 이 소식 보고 저도 놀랐는데요", "솔직히 이건 예상 못 했어요"
4. 1인칭 주관 표현: 본문 중간 삽입
   예: "솔직히", "이건 좀 의외였는데", "개인적으로는", "보면서 소름 돋았어요"
5. 테이블 1~2개로 제한 (테이블 남발 금지)
6. 맺음말 스타일 매번 교체: 응원형 / 질문형 / 요약형 / 다음편 예고형 중 택1
