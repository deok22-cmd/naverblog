# 티스토리 원고 수동 작성 가이드 (2026-05-20 신설)

> ※ **재개 완료(2026-05-20)** — 자동 모드는 `daily-prompt.md §5`로 정상 복원됨. 본 가이드는 사용자가 *수동 1건*(여행 vlog 톤)을 별도 요청할 때만 사용한다. 자동 5건과는 디자인·톤이 의도적으로 다름.

## 트리거 (사용자 발화 예)
- "○○ 지역 ○○ 느낌으로 티스토리 원고 작성해줘"
- "△△ 여행 후기 티스토리에 올릴 거 1건 써줘"

## 입력 (사용자가 매번 지정 / 미지정 시 한 번만 묻는다)
- **지역·테마·톤**: 필수
- **이미지 개수**: 사용자가 별도 지시 (기본 = 0개. 임의로 박지 않는다)
- **이미지 출처**: 사용자가 보유한 실제 여행 사진 경로 — **AI 생성 이미지·placeholder 박스 절대 금지**
- **분량**: 별 지시 없으면 본문 약 4,000~6,000자 (vlog 호흡 기준)
- **가족·맥락 정보**: 사용자가 알려준 배경은 톤·행간에 반영하되, "원고에 직접 쓰지 말라"고 한 부분은 *언급하지 않는다*.

## 출력
- 경로: `output_tistory/<YYMMDD>/<slug>.html` 1건만
- 네이버·인스타 등 다른 채널 동봉 X
- `국내여행지.md` 트래커 자동 마킹 X
- git commit/push는 사용자 요청 시에만

## 디자인 — 여행 vlog 톤 (자동 모드와 의도적으로 다름)

자동 모드 5건은 정보형 표준(Platinum v5: teal h1·번호 h2·intro-box·tip-box·info-table)을 따르지만, **수동 1건은 vlog 같은 자연스러운 호흡**이다. 정보 박스·번호 매김·강한 색 강조를 줄이고, **사진과 짧은 단락이 번갈아 흐르는 구조**를 사용한다.

### 외곽 래퍼 (유지)
```html
<div style="font-family: 'Apple SD Gothic Neo', sans-serif; font-size: 16px; line-height: 1.9; color: #222; max-width: 780px; margin: 0 auto; padding: 16px;">
```

### h1 — 차분한 검정 + 옅은 회색 밑줄
자동 모드의 teal 5px solid 강한 강조를 쓰지 않는다.
```html
<h1 style="font-size: 1.5em; font-weight: 700; color: #1a1a1a; padding-bottom: 10px; border-bottom: 1px solid #e0e0e0; margin: 0 0 6px 0;">제목</h1>
```

### (선택) 부제 한 줄
h1 바로 아래에 영문/짧은 부제. vlog 인트로 느낌.
```html
<p style="font-size: 0.88em; color: #999; letter-spacing: 1px; margin: 0 0 32px 0;">SHANGHAI · WITH DAUGHTERS</p>
```

### 인트로 — 박스 없이
자동 모드의 teal `이 글을 쓴 이유` 박스를 사용하지 않는다. 대신 **사진 한 장 + 짧은 두세 문장**으로 시작한다.

### h2 — 번호 없이, 가는 회색 선만
번호 prefix `01.` 사용하지 않는다. 짧은 풍경적 제목.
```html
<h2 style="font-size: 1.1em; font-weight: 600; color: #333; border-left: 3px solid #c0c0c0; padding-left: 12px; margin: 56px 0 18px 0;">소제목</h2>
```

### 본문 단락 — 짧은 호흡
1~3문장 단위로 끊는다. vlog 자막을 옮긴 듯한 호흡. `<br/><br/>`을 본문 안에 우겨넣지 않고, 별도 `<p>`로 단락을 분리.
```html
<p style="margin: 0.9em 0;">한두 문장. 짧게.</p>
<p style="margin: 0.9em 0;">다음 호흡.</p>
```

### 이미지 — 표준 1줄 블록, 짧은 자막 캡션
사용자 제공 경로 그대로. AI placeholder 박스 일체 금지. 캡션은 vlog 자막처럼 1~5단어.
```html
<div style="margin:36px 0;text-align:center;"><img alt="..." src="<사용자 제공 경로>" style="max-width:100%;height:auto;border-radius:6px;"/><div style="font-size:0.88em;color:#888;margin-top:8px;">짧은 자막</div></div>
```

### 표·info-table·tip-box — 거의 안 씀
정보 박스는 vlog 톤을 깬다. 사용자가 명시 요청 시만.

### 추천박스 — 단순 회색 라인
자동 모드의 흰 박스 형태 대신 가는 회색 구분선과 인라인 링크.
```html
<div style="border-top: 1px solid #e0e0e0; padding-top: 18px; margin-top: 48px; font-size: 0.92em; color: #666;">
같이 보면 좋은 글<br/>
<a href="..." style="color: #555; text-decoration: none;">제목 1</a> · <a href="..." style="color: #555; text-decoration: none;">제목 2</a>
</div>
```

### 태그칩 — 작고 회색
자동 모드의 진한 회색 박스 칩 대신 가벼운 인라인.
```html
<div style="margin-top: 24px; font-size: 0.82em; color: #999;">
<span style="margin-right: 10px;">#태그1</span><span style="margin-right: 10px;">#태그2</span>...
</div>
```
10개 권장(검색용).

### 어조
- 존대말이되 살짝 가볍게 (`~했어요`/`~였습니다` 자연스럽게 섞임).
- 감정·장면 위주, 정보 나열 톤 지양.
- 사용자가 알려준 가족·맥락 정보 중 "원고에 쓰지 말라"고 한 부분은 *언급하지 않되 행간에 무게로 깔아둔다*.

## 게이트
**수동 vlog 디자인은 자동 게이트(`.scripts/tistory-gate.ps1`) F1~F8과 의도적으로 어긋난다** (특히 F2 h1 5px·F8 태그칩 형식). 따라서 **수동 1건은 게이트 PASS 의무 없음** — 사용자가 직접 검수한다. 자동 모드 5건은 그대로 게이트 강제. 수동 글이 같은 폴더에 있을 때 게이트가 FAIL을 표시할 수 있는데, 그건 의도된 차이이고 push 차단 사유 아님(수동 push는 사용자가 별도 수행).

## 자동 재개 절차
사용자가 *"티스토리 원고 자동작성 재개해줘"* 라고 요청하면 즉시 다음 3가지 동시 복원:
1. `daily-run.ps1` 상단 `$TistorySuspended = $false`
2. `.scripts/daily-prompt.md` 큰 틀 줄·§5 헤더의 SUSPENDED 배너 제거
3. 본 파일 상단에 `> ※ 재개 완료(YYYY-MM-DD) — 본 가이드는 보존만, 자동 모드는 daily-prompt §5 사용.` 추가
