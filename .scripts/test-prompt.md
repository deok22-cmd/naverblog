# Naverblog Daily Pipeline — 2026-05-02 강제 테스트 실행

본 프롬프트는 사용자가 "5월 2일 원고 작업을 테스트로 진행"을 명시 요청한 1회성 실행이다. 시스템 날짜는 2026-05-01이지만 **본 작업의 모든 처리에서 YYMMDD를 `260502`로, 풀 날짜를 `2026-05-02`로 강제 사용**한다. 이는 내일 04:00 자동 실행이 수행할 작업을 미리 검증하기 위함이다.

## 핵심 지시

`.scripts/daily-prompt.md`의 전체 절차를 그대로 수행하되, 다음만 오버라이드한다.

- `YYMMDD = 260502`
- 풀 날짜 = `2026-05-02`
- 직전일 = `260501` (내부 링크 후보 추출 대상)
- `Naverblog.md` 4+1 Rule, Platinum v5 표준, Idempotent Resume 모드, 트래커 갱신 등은 **그대로 적용**

## 예상 동작 (참고용)

- `output/260502/index.html` 이미 존재 → **Resume 모드** 진입
- 대시보드 파싱 시 5개 행:
  - #1 임실 치즈테마파크 → ✔️ 작성 완료 (보존, 절대 손대지 않음)
  - #2 전주 국제영화제 → ⏳ (작성 대상)
  - #3 여주 도자기축제 → ⏳ (작성 대상)
  - #4 밀양 아리랑대축제 → ⏳ (작성 대상)
  - #5 FILTER & SORT 함수 → ⏳ (작성 대상)
- 4건의 원고 작성 (전부 placeholder 모드, `<img>` 태그 절대 미사용)
- 작성 완료 후 대시보드의 4개 ⏳ 행을 ✔️로 갱신
- `국내여행지.md`: #86~89를 `작성완료(05.02)`로 갱신 (#86은 임실로 이미 발행 완료, #87~89는 본 실행에서 작성)
- `sub_topic_tracker.md`: 히스토리에 `2026-05-02 | SPREADSHEET | FILTER & SORT 함수` 행 추가, `NEXT_TOPIC`을 `RECIPE`로 토글, `LAST_UPDATED`를 `2026-05-02`로 갱신
- `spreadsheet.md`: FILTER & SORT 항목 상태를 `✅ 완료`로, 작성일을 `2026-05-02`로 갱신
- git 명령 절대 미실행
- 종료 시 `[DAILY OK 2026-05-02] ...` 한 줄 출력
