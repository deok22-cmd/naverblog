구글 스프레드시트 IMPORTRANGE 함수 완벽 가이드: 데이터 동기화의 핵심

구글 스프레드시트를 실무에서 사용할 때 가장 빈번하게 발생하는 니즈 중 하나는 "서로 다른 파일에 흩어진 데이터를 하나로 모으는 것"입니다. 엑셀과는 달리 구글 시트는 웹 기반이기에 IMPORTRANGE라는 강력한 함수를 통해 별개의 파일 간에도 실시간 데이터 연결이 가능한데요. 오늘 포스팅에서는 이 함수의 기본 문법부터 실무 활용 팁, 그리고 권한 오류 해결 방법까지 상세히 다루어 보겠습니다.

1. IMPORTRANGE 함수란 무엇인가?

IMPORTRANGE는 특정 스프레드시트 파일의 데이터를 다른 스프레드시트 파일로 불러올 때 사용하는 함수입니다. 파일이 물리적으로 나누어져 있어도 URL(주소)만 있다면 원하는 범위의 데이터를 실시간으로 동기화할 수 있다는 것이 최대 장점입니다.

이 함수를 활용하면 '매출 관리 시트'의 데이터를 '종합 대시보드 시트'로 자동으로 불러오거나, 팀별로 작성한 업무 리스트를 관리자 시트로 취합하는 작업 등을 자동화할 수 있습니다. 수동으로 복사-붙여넣기 하던 비효율적인 업무 방식을 획기적으로 개선해 주는 툴입니다.
서로 다른 시트를 연결해 주는 실시간 데이터 동기화 기술 / AI 제작 이미지

2. IMPORTRANGE 함수의 기본 문법과 인수 설명

이 함수의 문법은 의외로 간단합니다. 하지만 인수를 입력할 때 반드시 큰따옴표("")를 사용해야 한다는 점을 잊지 마세요. 기본 구조는 다음과 같습니다.

=IMPORTRANGE("스프레드시트_URL", "시트이름!범위")
첫 번째 인수인 URL은 데이터를 가져올 원본 파일의 주소 전체를 입력하거나, 주소 중간의 고유 ID값만 입력해도 무방합니다. 두 번째 인수는 원본 파일 내의 시트 이름과 범위(예: A1:E100)를 지정합니다. 시트 이름을 생략할 경우 첫 번째 시트의 범위를 참조하게 됩니다.

3. 실전 예제: 매출 데이터를 통합 시트로 가져오기

예를 들어, 원본 파일의 주소가 `https://docs.google.com/spreadsheets/d/ABCDE12345/edit`이고, 그 안의 '4월매출' 시트에서 A1부터 D50까지의 데이터를 가져오고 싶다면 다음과 같이 입력합니다.

=IMPORTRANGE("https://docs.google.com/spreadsheets/d/ABCDE12345/edit", "4월매출!A1:D50")
수식을 입력하고 처음 엔터를 치면 #REF! 오류가 뜰 수 있습니다. 이는 두 파일 간의 연결 권한이 없기 때문인데요. 해당 셀을 클릭하면 나타나는 [액세스 허용] 버튼을 한 번만 눌러주면 이후부터는 데이터가 정상적으로 표시됩니다.
정확한 수식 입력으로 데이터 연결 권한을 획득하는 과정 / AI 제작 이미지

4. IMPORTRANGE 활용 시 주의해야 할 성능 이슈

강력한 함수이지만 남발할 경우 스프레드시트의 속도 저하를 불러올 수 있습니다. 특히 한 시트 내에서 수십 개의 IMPORTRANGE를 사용하는 경우, 매번 웹 서버를 통해 데이터를 새로 고침하기 때문에 로딩 시간이 길어지게 됩니다.

성능을 최적화하려면 가급적 큰 범위를 한 번에 불러오는 것이 좋습니다. 개별 셀마다 함수를 쓰는 대신, 한 번의 수식으로 데이터 덩어리를 가져온 뒤 내부에서 FILTER나 QUERY 함수로 재가공하는 방식을 추천합니다. 또한 원본 시트의 구조가 자주 바뀌면 참조 범위가 어긋날 수 있으니 주의가 필요합니다.

5. QUERY 함수와 결합하여 필요한 데이터만 골라내기

IMPORTRANGE로 가져온 방대한 데이터 중 특정 조건에 맞는 데이터만 필터링하고 싶다면 QUERY 함수와의 결합이 필수적입니다. 수식 한 줄로 데이터 추출과 필터링을 동시에 해결할 수 있습니다.

=QUERY(IMPORTRANGE("URL", "범위"), "select * where Col3 > 100000")
이때 주의할 점은 QUERY 내부에서 열을 지정할 때 A, B, C 대신 Col1, Col2, Col3 형식을 사용해야 한다는 것입니다. 외부 데이터를 참조하는 배열 형태이기 때문인데요. 이 조합만 익혀두셔도 실무 데이터 관리의 80% 이상은 자동화가 가능합니다.

6. 동적 범위 지정을 위한 INDIRECT 활용법

불러올 시트 이름이나 범위가 수시로 변하는 경우, INDIRECT 함수를 함께 사용하면 더욱 유연한 설계가 가능합니다. 예를 들어 A1 셀에 '5월매출'이라고 입력하면 자동으로 해당 시트의 데이터를 가져오게 만들 수 있습니다.

수식 내에서 문자열 조합을 통해 주소를 생성하면, 매번 수식을 수정할 필요 없이 특정 셀의 값만 바꿔주는 것으로 전체 대시보드를 업데이트할 수 있습니다. 이는 대규모 프로젝트 관리나 연간 보고서 작성 시 매우 유용한 고급 스킬입니다.
IMPORTRANGE로 취합된 데이터를 활용한 전문 대시보드 구성 / AI 제작 이미지

7. 데이터 보안 및 공유 권한 관리 전략

IMPORTRANGE로 연결된 파일은 원본 파일에 대한 읽기 권한이 있는 사용자만이 연결을 설정할 수 있습니다. 한 번 연결이 완료되면, 결과 시트를 공유받은 다른 사용자들은 원본 파일에 대한 권한이 없더라도 취합된 데이터는 볼 수 있게 됩니다.

이러한 특성을 활용하면 원본의 민감한 데이터(개인정보 등)는 숨기고, 필요한 요약 데이터만 추출하여 팀원들에게 공유하는 '보안 브릿지' 역할을 수행할 수 있습니다. 단, 결과 시트를 공유할 때 원본의 핵심 로직이 노출되지 않도록 범위 설정을 세밀하게 하는 것이 중요합니다.

8. 자주 발생하는 오류 유형과 해결 방법

가장 흔한 #REF! 오류는 앞서 설명한 액세스 허용 문제나 주소 오타인 경우가 대부분입니다. 만약 #VALUE! 오류가 뜬다면 인수에 큰따옴표가 빠지지 않았는지 확인해 보세요.

또한 원본 시트가 삭제되었거나 주소가 변경된 경우에도 오류가 발생합니다. 가끔 '결과가 너무 큽니다'라는 오류가 뜨기도 하는데, 이는 구글 시트의 단일 셀 처리 한도를 넘었을 때 발생하므로 불러오는 범위를 나누거나 데이터 정제 후 가져오는 방식으로 해결해야 합니다.

9. 맺음말: IMPORTRANGE로 스마트 워크 실현하기

지금까지 구글 스프레드시트의 핵심 기능인 IMPORTRANGE 함수에 대해 자세히 알아보았습니다. 처음에는 생소할 수 있지만, 한 번 세팅해 두면 매일 반복되는 수동 작업 시간을 획기적으로 줄여주는 고마운 함수입니다.

데이터가 여기저기 흩어져 있어 고통받고 계셨다면, 오늘 배운 내용을 바탕으로 나만의 자동 통합 시스템을 구축해 보시는 건 어떨까요? 작은 차이가 업무의 퀄리티를 바꿉니다. 궁금하신 점은 댓글로 남겨주시면 성심껏 답변해 드리겠습니다!
