📌 1. UNIQUE, SORTN 함수를 알면 데이터 정제가 달라진다

데이터 업무에서 가장 자주 마주치는 두 가지 난관이 있습니다. 첫째는 중복 데이터이고, 둘째는 특정 기준 상위/하위 N개 추출입니다. 이 두 가지 문제를 해결하지 못하면 CRM 고객 명단, 매출 분석 보고서, 재고 관리 시트 등 거의 모든 업무 데이터가 엉망이 됩니다.

과거에는 이 두 작업을 처리하기 위해 데이터 > 중복 항목 삭제 메뉴를 수동으로 클릭하거나, 복잡한 COUNTIF 수식으로 보조 열을 만들어 필터링하는 방법을 사용했습니다. 이 방식은 원본 데이터가 업데이트될 때마다 반복 작업이 필요하고, 실수 가능성도 높아 대용량 데이터 환경에서는 한계가 분명했습니다.

구글 스프레드시트는 이 문제를 해결하기 위해 UNIQUE와 SORTN이라는 두 가지 강력한 내장 함수를 제공합니다. 이 두 함수는 별도의 매크로나 스크립트 없이 순수 수식만으로 중복 제거와 랭킹 추출을 동적으로 처리합니다. 원본 데이터가 변경되면 결과도 실시간으로 자동 갱신되므로, 한 번 수식을 설정해두면 추가 작업이 전혀 필요 없는 완전한 자동화가 실현됩니다.

🔤 2. UNIQUE 함수 기본 문법 & 작동 원리

UNIQUE 함수는 범위 내에서 고유한(중복되지 않는) 행 또는 열만 추출해 반환하는 함수입니다. 가장 직관적인 중복 제거 도구로, 복잡한 COUNTIF 보조 수식 없이 한 줄로 중복을 제거할 수 있습니다.

기본 문법:
=UNIQUE(범위, [열 기준 여부], [정확히 한 번만 나타나는 값])
각 인수의 의미를 하나씩 살펴봅시다.

① 범위 (필수): 중복을 제거할 셀 범위입니다. 단일 열(예: A2:A100)부터 다중 열 범위(예: A2:C100)까지 모두 지원합니다.
② 열 기준 여부 (선택, 기본값 FALSE): FALSE이면 행 기준으로 중복 제거(기본 동작), TRUE이면 열 기준으로 중복 제거합니다. 대부분의 실무에서는 기본값인 FALSE를 사용합니다.
③ 정확히 한 번만 나타나는 값 (선택, 기본값 FALSE): TRUE로 설정하면 데이터 중 딱 한 번만 등장하는 항목만 반환합니다. 중복 항목을 완전히 제외하고 싶을 때 사용합니다.

가장 단순한 활용 예시를 살펴보겠습니다. A열에 지역명이 반복되어 입력되어 있을 때, 고유한 지역명 목록만 추출하고 싶다면 다음과 같이 입력합니다.
=UNIQUE(A2:A100)
이 수식은 A2:A100 범위에서 중복된 지역명을 제거하고 각 지역명을 딱 한 번씩만 나열한 결과를 반환합니다. 원본 데이터에 새로운 지역명이 추가되면 결과도 즉시 반영됩니다. 이것이 수동 중복 삭제와의 결정적인 차이입니다.

🧹 3. UNIQUE 활용: 중복 제거 실전 사례

이론보다 실전 예시가 이해에 훨씬 빠릅니다. 아래 시나리오를 통해 UNIQUE 함수의 실무 활용법을 구체적으로 익혀보겠습니다.

[시나리오 1: 고객 주문 데이터에서 고유 고객 목록 추출]
A열 (고객명)B열 (주문 금액)
김민준35,000원
이서연12,000원
김민준68,000원
박지호25,000원
이서연44,000원
위와 같은 주문 데이터에서 고유 고객 목록만 추출하려면 E2 셀에 아래 수식을 입력합니다.
=UNIQUE(A2:A6)
결과는 김민준, 이서연, 박지호 3개의 고유 이름이 반환됩니다. 이 결과 목록에 SUMIF를 조합하면 각 고객의 총 주문 금액도 자동으로 집계할 수 있습니다.
=SUMIF(A2:A6, E2#, B2:B6)
여기서 E2#는 E2부터 시작하는 UNIQUE 결과 배열 전체를 참조하는 '배열 확장 연산자'입니다. 이 수식 조합 하나로 고객별 총 주문 금액 집계표가 자동으로 완성됩니다.

[시나리오 2: 여러 열을 기준으로 한 복합 중복 제거] 예를 들어 A열(날짜), B열(담당자명), C열(거래처명)이 있는 데이터에서 날짜+담당자 조합이 중복된 행을 제거하고 싶다면 다음과 같이 입력합니다.
=UNIQUE(A2:B100)
이 수식은 A열과 B열을 함께 비교해 두 값이 모두 동일한 행이 있을 경우 중복으로 처리하여 한 번만 반환합니다. 다중 열 기준 중복 제거가 수식 한 줄로 해결되는 것입니다.

🔬 4. UNIQUE 심화: 조건부 중복 제거 & 세 번째 인수 활용

UNIQUE 함수의 세 번째 인수인 '정확히 한 번만 나타나는 값'을 활용하면 더 정교한 중복 제거가 가능합니다. 이 인수를 TRUE로 설정하면, 두 번 이상 등장하는 데이터는 결과에서 완전히 제외되고 딱 한 번만 등장하는 항목만 반환됩니다.

실전 활용 사례: 이벤트 참가 신청 명단에서 단 한 번만 신청한 참가자를 추출해야 하는 경우(중복 신청자 제외, 정상 신청자만 추출)에 사용합니다.
=UNIQUE(A2:A100, FALSE, TRUE)
이 수식은 A2:A100 범위에서 딱 한 번만 등장하는 값만 반환합니다. 두 번 이상 신청한 참가자는 결과에서 제외되어, 순수하게 1회만 신청한 참가자 목록이 만들어집니다.

⚠️ 주의사항: UNIQUE 함수는 텍스트 대소문자를 구별하지 않습니다. 즉, "서울"과 "서울"은 동일하게 처리되지만, "Apple"과 "apple"은 서로 다른 값으로 인식됩니다. 영문 데이터를 다룰 때는 LOWER() 또는 UPPER() 함수로 대소문자를 통일한 후 UNIQUE를 적용하세요.

🏆 5. SORTN 함수 기본 문법 & 작동 원리

SORTN 함수는 범위를 정렬한 후 상위 또는 하위 N개의 행만 반환하는 함수입니다. 단순히 N번째 값 하나만 찾는 LARGE/SMALL 함수와 달리, SORTN은 상위 N개 전체 행 데이터를 배열로 반환하기 때문에 활용 범위가 훨씬 넓습니다.

기본 문법:
=SORTN(범위, [n], [표시 타이], [정렬 기준 열], [오름차순 여부])
① 범위 (필수): 정렬 및 추출할 데이터 범위입니다.
② n (선택, 기본값 1): 반환할 행의 수입니다. 예를 들어 5를 입력하면 상위 5개 행이 반환됩니다.
③ 표시 타이 (선택, 기본값 0): 동점(타이) 처리 방식을 지정합니다.
    - 0: n번째까지만 엄격하게 반환 (동점이어도 n개 초과 불가)
    - 1: n번째와 동점인 항목을 모두 포함 (n개를 초과할 수 있음)
    - 2: 중복 제거 후 상위 n개 반환
    - 3: 중복 제거 후 n번째와 동점인 항목 모두 포함
④ 정렬 기준 열 (선택, 기본값 1): 정렬 기준이 되는 열 번호입니다.
⑤ 오름차순 여부 (선택, 기본값 TRUE): FALSE이면 내림차순(큰 값부터), TRUE이면 오름차순(작은 값부터) 정렬됩니다.

가장 기본적인 활용 예시: A열(직원명), B열(매출액) 데이터에서 매출 상위 3명을 추출하려면:
=SORTN(A2:B100, 3, 0, 2, FALSE)
이 수식은 B열(2번째 열)을 기준으로 내림차순 정렬한 후 상위 3행을 반환합니다. 결과로 직원명과 매출액이 함께 반환되어 보고서에 바로 활용할 수 있습니다.

📊 6. SORTN 활용: 상위/하위 N개 추출 실전 사례

SORTN의 강력함은 실전 사례에서 더욱 빛납니다. 아래 두 가지 시나리오를 통해 실무 활용법을 익혀보겠습니다.

[시나리오 1: 월별 판매 데이터에서 판매량 TOP 5 상품 추출] A열(상품명), B열(판매 수량), C열(판매 금액) 데이터가 있을 때:
=SORTN(A2:C100, 5, 0, 2, FALSE)
이 수식은 B열(판매 수량) 기준으로 내림차순 정렬 후 상위 5개 행(상품명+판매수량+판매금액)을 반환합니다. 헤더 포함 결과를 원한다면 VSTACK으로 헤더를 붙일 수 있습니다.

[시나리오 2: 동점 처리가 중요한 성적 분석] 성적 데이터에서 상위 3위까지 추출할 때 동점자가 있으면 모두 포함시켜야 하는 경우:
=SORTN(A2:B30, 3, 1, 2, FALSE)
타이 인수를 1로 설정하면, 예를 들어 3위와 동점인 학생이 2명일 경우 총 4행이 반환됩니다. 성적표, 순위 집계, 경쟁사 분석 등에서 동점자를 누락하지 않고 공정하게 처리할 때 매우 유용합니다.

[시나리오 3: 최저 재고 상품 알림 시스템] 재고 관리 시트에서 재고가 가장 적은 상품 5개를 추출해 자동으로 발주 대상 목록을 만들 수 있습니다:
=SORTN(A2:D100, 5, 1, 3, TRUE)
C열(재고 수량)을 기준으로 오름차순 정렬 후 하위 5개(재고 가장 적은 5개) 행을 반환합니다. 이 수식 하나로 재고 부족 상품 파악이 실시간으로 자동화됩니다.

⚡ 7. UNIQUE + SORTN 조합: 강력한 데이터 정제 파이프라인

UNIQUE와 SORTN을 함께 쓰면 '중복 제거 후 상위 N개 추출'이라는 실무에서 매우 자주 필요한 작업을 한 번에 처리할 수 있습니다. 두 함수의 조합은 배열 함수의 특성상 자연스럽게 중첩 사용이 가능합니다.

[실전 예시: 중복 제거 후 상위 5개 고객 추출] 고객별 여러 건의 주문이 섞인 데이터에서, 먼저 고객별로 중복을 제거하고 그 다음 총 구매액 기준 상위 5명을 추출하는 경우입니다.
/* 1단계: 고객별 총 구매액 집계 (보조 범위 E:F에 생성) */ =UNIQUE(A2:A100) /* E2: 고유 고객 목록 */ =SUMIF(A2:A100, E2#, B2:B100) /* F2: 고객별 합산 금액 */ /* 2단계: 집계 결과에서 상위 5명 추출 */ =SORTN(E2#, 5, 0, 2, FALSE)
이 파이프라인은 원본 데이터가 아무리 복잡하고 중복이 많아도 항상 정확한 고객별 집계 후 상위 5명을 자동으로 추출합니다. 실무에서 CRM 분석, VIP 고객 선정, 성과 보고서 작성에 바로 활용할 수 있는 완결형 수식 구조입니다.

⚖️ 8. FILTER, QUERY와의 비교 & 성능 분석

중복 제거와 상위 N개 추출 작업에서 UNIQUE·SORTN 외에도 FILTER나 QUERY 함수를 활용하는 방법도 있습니다. 각 함수의 특성을 비교해 상황에 맞는 최적의 도구를 선택하는 것이 중요합니다.

기준UNIQUESORTNFILTERQUERY
주요 목적중복 제거상위 N개 추출조건 필터링SQL 방식 복합 처리
문법 난이도매우 쉬움쉬움보통어려움

다중 조건제한적제한적강력함매우 강력함
성능(대용량)빠름빠름보통느림
적합한 상황단순 중복 제거랭킹 추출조건 기반 추출복잡한 집계 분석

결론적으로, 단순한 중복 제거는 UNIQUE, 상위/하위 N개 추출은 SORTN이 가장 직관적이고 빠릅니다. 복잡한 조건이 결합될 때는 FILTER나 QUERY를 선택하되, 수식이 길어질수록 성능 저하와 오류 가능성이 높아지므로 최대한 단순하게 구성하는 것이 실무의 정석입니다. 대용량 데이터(10만 행 이상)에서는 QUERY보다 UNIQUE+SORTN 조합이 처리 속도 면에서 유리한 경우가 많습니다.

📈 9. 실전 대시보드: UNIQUE & SORTN으로 만드는 자동화 보고서

이번 섹션에서는 UNIQUE와 SORTN을 모두 활용해 완전 자동화된 실무 대시보드를 구축하는 방법을 소개합니다. 아래 예시는 팀별 월간 매출 실적 보고서를 자동화하는 구조입니다.

[데이터 구조 가정] A열: 날짜, B열: 담당자명, C열: 팀명, D열: 매출액

STEP 1. 고유 팀 목록 자동 추출
=UNIQUE(C2:C500)
→ F2에 입력하면 팀명 목록이 자동으로 생성되며, 새로운 팀이 추가되면 즉시 반영됩니다.
STEP 2. 팀별 총 매출 집계
=SUMIF(C2:C500, F2#, D2:D500)
→ G2에 입력하면 F열의 팀별 매출이 자동 집계됩니다.
STEP 3. 매출 TOP 3 팀 자동 추출
=SORTN(HSTACK(F2#, G2#), 3, 0, 2, FALSE)
→ F·G열의 팀명+매출 배열을 HSTACK으로 합친 후 매출 기준 상위 3개 팀을 추출합니다.
STEP 4. 전월 대비 성장률 자동 계산 (선택)
→ 이 단계에서는 ARRAYFORMULA와 조합해 전월 데이터와의 비율을 자동 계산하면 완전한 자동화 월간 보고서가 완성됩니다.

이처럼 UNIQUE와 SORTN은 단독으로도 강력하지만, SUMIF·HSTACK·ARRAYFORMULA 등 다른 함수들과 조합할 때 진정한 '데이터 자동화 파이프라인'으로 진화합니다. 오늘 배운 내용을 바탕으로 여러분만의 스마트한 업무 자동화 대시보드를 만들어 보세요!

📌 오늘의 핵심 요약
✅ UNIQUE: 범위에서 중복을 제거해 고유한 값만 반환. 세 번째 인수 TRUE로 1회만 등장하는 값만 추출 가능.
✅ SORTN: 범위를 정렬해 상위 N개 행 반환. 타이 처리 인수로 동점자 포함 여부 제어 가능.
✅ 두 함수 조합: UNIQUE로 중복 제거 → SORTN으로 랭킹 추출의 순서가 실무 데이터 정제의 황금 파이프라인.
