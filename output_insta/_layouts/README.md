## 0 | 절대 원칙 (전 레이아웃 공통)
- 캔버스 `viewBox="0 0 1080 1350"`, `width/height` 동일.
- 배경 슬롯: `<rect id="BG__REPLACE_WITH_IMAGE" .../>` **정확히 1개** + 위/아래 안내 주석 (Phase B·Figma 공용).
- 폰트 `Pretendard, 'Apple SD Gothic Neo', sans-serif`, 한글은 살아있는 `<text>` (path 변환 금지).
- 카테고리 팔레트(여행 `#00796b`/`#004d40`/`#e0f2f1`/보조 `#80cbc4`/`#b2dfdb`/`#cfd8d6`) — 카테고리만 치환.
- 좌측 `brand-bar`(14px) 유지.
- **사진이 주인공 (Hard Rule)**: 가독성 확보는 **오직 그라데이션 스크림(linearGradient)**으로만 한다. 불투명 솔리드 밴드, 큰 불투명 패널, 보더 박스, 사진을 가리는 사각형은 **전면 금지**. 어떤 배경을 넣어도 사진이 비쳐 보여야 한다.
  - **유일한 명문 예외 (2026-05-18, 사용자 승인 전면 표준)**: **G9 CTA 카드의 흰색 둥근 '팔로우 카드' 1개**(아바타+`@핸들`+팔로우 버튼+커서, `G9_cta_follow.svg`의 `#follow-card`). 전환 목적의 의도된 표준 컴포넌트다. **CTA(마지막 카드) 외에는 절대 적용 금지**, 자동검수·후속 에이전트가 이 흰 카드를 §0 위반으로 보고 되돌리지 말 것. CTA 카드도 배경 사진은 헤드라인 위/아래로 여전히 비쳐 보여야 한다.
- **넘버링·배지 금지 (Hard Rule)**: 대형 워터마크 숫자, 원형/사각 번호 배지, "01 ·" 식 카드번호 키커, 디자인 장식용 거대 숫자/스탯 타일 — 전부 금지. 위계는 **텍스트 크기·굵기·색·여백(타이포그래피)**으로만 만든다. (항목 마커가 필요하면 숫자 대신 얇은 강조 틱 `rect` 하나만 — 타이포 수준)

## 2 | 가변 길이 카드 세트 (2026-05-18 개정 — 천편일률 금지)

> **핵심 원칙**: 모든 글을 똑같이 10장·동일 순서로 찍어내지 않는다. 커버·CTA만 고정 북엔드, **중간 카드는 원고 내용에 따라 개수·역할·순서가 글마다 달라진다.**

**카드 총수 = 가변 7~10장** (커버 1 + 중간 5~8 + CTA 1). **최소 7장 보장** — 콘텐츠가 가벼워도 원고 h2 `point`를 더 넣어 7장 이상을 채운다(상한 10).

**역할 풀(중간 카드 후보)** — 원고에 해당 소스가 있을 때만 후보가 된다:
| 역할키 | 소스 | 허용 레이아웃 풀 | 비고 |
|---|---|---|---|
| `cover` | h1+intro | **G0 고정** | 항상 1번 |
| `why` | intro 3줄 | `[B, C, E]` | 거의 항상 포함(훅) |
| `point` ×2~5 | h2 대표 꼭지 | `[A, B, C, D, E, F]` | 원고 꼭지 수·중요도로 2~5개(7장 하한 보충용으로 가감) |
| `info` | info-table | `[P1(행≥5) / P4(핵심지표 3~4)]` | 표/지표 있을 때만 |
| `course` | step-box | `[P2 / 단계≤3이면 F]` | 동선·순서 있을 때만 |
| `tips` | tip-box | `[P3 / 3포인트면 F]` | 저장 유도, 꿀팁 있을 때만 |
| `caution` | caution-box | `[A, E]` | 주의 있을 때만 |
| `cta` | recommend+저장유도 | **G9 고정** | 항상 마지막 |

## 3 | 결정 규칙 (우선순위 순)

1. **콘텐츠 형태가 최우선(하드 제약)**: label↔value 표=P1, 핵심지표 3~4=P4, 순차 단계=P2, 체크리스트=P3, 정확히 3개 병렬=F, 단일 핵심 메시지=D. 소스가 없으면 그 역할(info/course/tips/caution) 카드는 **만들지 않는다**(없는 수치·장소·평점 창작 절대 금지). 단 **총수 하한은 7장** — 박스류가 부족해도 원고에 실재하는 h2 꼭지를 `point` 카드로 더 만들어 7장을 채운다(여행 원고는 h2가 9~10개라 항상 충분). 즉 "짧아지는 사유"는 7장까지만 허용, 그 아래로는 내려가지 않는다.
2. **글(slug)별 길이·구성 시드** — 글마다 장수와 스켈레톤이 달라지게:
   - `seed = (slug 문자열의 각 문자 코드 합)`
   - **중간 카드 수** `M = 5 + (seed % 4)` → 5~8 (총 7~10장). 하한 5(=총 7) 미만으로 내려가지 않는다 — 다른 역할 소스가 부족하면 `point`를 추가해 M을 채운다.
   - **역할 선택 우선순위**: `why` → `point`(중요 꼭지 순) → `info` → `course` → `tips` → `caution`. M개가 찰 때까지 이 순서로 채우되, `point` 개수는 `2 + (seed % 3)`개(2~4)로 글마다 다르게 시작하고, 다른 역할 소스가 모자라 M이 안 차면 `point`를 더 추가(원고 h2 순서대로)해 반드시 M≥5를 만족시킨다. 우선순위 뒤쪽(`caution`·잉여 `tips`)이 M 초과 시 잘린다.
   - **순서(스켈레톤) 셔플**: 선택된 중간 역할들의 배열을 `seed`로 회전시킨다 — 고정 순서(why→point→info→course→tips→caution) 금지. 단 `why`는 커버 직후(2번), `tips`(저장 유도)는 가능하면 CTA 직전 근처에 둔다. 그 외 `point`/`info`/`course`/`caution`은 seed로 위치 교차.
3. **레이아웃 타입 회전**(선택된 카드의 형식):
   - 콘텐츠 풀 `POOL=[A,B,C,D,E,F]`, `start = seed % 6`. `point` 카드들은 `POOL[start], POOL[(start+2)%6], POOL[(start+4)%6]` 순으로 서로 다른 타입.
   - `why = [B,C,E][seed%3]`(첫 `point`와 다르게), `caution = [A,E][seed%2]`.
   - 콘텐츠 형태 제약(규칙 1)이 회전 결과를 덮어쓴다.
4. **인접 비동일**: 카드 N과 N+1은 같은 그룹타입 금지(커버·CTA·P타입은 고유라 자동 충족). 충돌 시 해당 카드 타입을 풀 내 다음 후보로 +1.
5. **결과 검증**: 같은 날 5개 슬러그의 (장수, 역할 순서) 튜플이 **최소 3종 이상 서로 달라야** 한다. 전부 같으면 시드 적용 실패 → 재배치.

> 수동·부분 수정 시에도 본 §2–3을 따른다. **범위는 7~10장(하한 7 엄수, 상한 10)**: 콘텐츠가 가벼우면 7장 근처, 풍부하면 최대 10장. 없는 사실(수치·장소·평점) 창작은 여전히 금지하되, 원고에 실재하는 h2를 `point`로 활용해 **최소 7장은 채운다**(6장 이하 금지).

- **§0의 두 하드룰을 절대 위반 금지**: 새 불투명 사각형/밴드/박스 추가 금지, 숫자 배지·워터마크·카드번호 키커·장식 거대숫자 추가 금지. 키커는 숫자 없는 짧은 단어(지명·주제)만.
- 파일명 `card_NN_<역할키워드>.svg` (NN=카루셀 위치, 01부터 연속·빈 번호 없음). `card_01`=G0, **마지막 카드**=G9(가변 길이라 항상 card_10이 아님).

- **CTA(G9) = 흰 팔로우 카드 표준 (2026-05-18 전면 교체, 사용자 승인)**: 마지막 카드는 **반드시 `G9_cta_follow.svg` 구조를 그대로 차용**한다(좌표·스크림·`#follow-card`·brand-bar·BG슬롯·viewBox 불변, 색만 카테고리 팔레트 치환). 필수 고정 요소: ① 상단 브랜드 워드마크 `travelkorea_365` ② 중앙 3줄 헤드라인 ③ 흰색 둥근 **팔로우 카드**(아바타+`@travelkorea_365`+팔로우 버튼+커서) ④ 서브 카피 한 줄 ⑤ info-chip(원고 위치/접근 사실, 없으면 그 g 삭제) ⑥ 블로그 안내 줄 ⑦ 해시태그 ⑧ 북마크 아이콘(저장 보강).

## 5 | 배경 프롬프트 가이드 (prompts.md 작성 시)
레이아웃이 회전하므로 특정 텍스트 영역을 비우라는 지시(예: "하단 1/3 비움")는 쓰지 않는다. 대신 **전체적으로 분위기 있고 과하지 않게 균일, 세로 4:5, 글자 없음**을 유지(스타일 앵커의 `no text...`는 그대로). 각 레이아웃이 자체 그라데이션 스크림으로 대비를 확보하므로 배경은 범용이어도 안전하다.
