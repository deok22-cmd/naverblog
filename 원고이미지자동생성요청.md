# 🖼️ 원고 이미지 일괄 자동 복원 프로토콜 (Batch Image Recovery)

본 문서는 사용자가 외부에서 미리 생성해둔 원고(HTML) 내의 플레이스홀더를 인식하여, AI 이미지를 일괄 생성하고 원고를 완성하는 자동화 프로세스를 정의합니다.

---

### 1. 전제 조건 (Prerequisites)
- **파일 위치**: 원고 파일이 `d:\lightsail\naverblog\output\yymmdd\` 폴더 내에 존재해야 함.
- **플레이스홀더 구조**: 각 HTML 파일 내에 아래와 같은 형식의 `.img-placeholder` 요소가 포함되어 있어야 함.
  ```html
  <div class="img-placeholder">
    <div class="ph-head">
      <span class="ph-file">images/yymmdd/파일명.png</span>
    </div>
    <pre class="prompt-text">영문 이미지 생성 프롬프트...</pre>
  </div>
  ```

---

### 2. 실행 명령 (Trigger Command)
에이전트에게 다음과 같이 요청하면 프로세스가 시작됩니다.
- **"오늘(또는 특정날짜) 생성된 모든 원고의 이미지 복원해줘"**
- **"해당 폴더의 플레이스홀더를 실제 이미지로 교체해줘"**

---

### 3. 에이전트 수행 단계 (Agent Workflow)
1.  **대상 스캔**: 지정된 날짜 폴더 내의 모든 HTML 파일을 읽습니다.
2.  **데이터 추출**: 각 파일에서 저장 경로(`.ph-file`)와 프롬프트(`.prompt-text`)를 추출합니다.
3.  **이미지 생성**: 추출된 프롬프트를 사용하여 1024x1024 (1:1 비율) 이미지를 생성합니다.
4.  **파일 관리**: 생성된 이미지를 프로젝트 루트 기준 지정된 경로(예: `d:\lightsail\naverblog\images\yymmdd\`)로 **`move`** 합니다.
5.  **코드 복원**: HTML 내의 `.img-placeholder` 전체 블록을 실제 `<img>` 태그로 교체합니다.
    - 예: `<img src="../../images/yymmdd/파일명.png" alt="캡션명">`
    - 이때 `alt` 속성은 바로 아래의 `.img-caption` 텍스트를 참조하여 채웁니다.

---

### 4. 주의 사항 및 팁
- **쿼터 관리**: 이미지 생성 도구의 일일 쿼터(약 15장 내외)를 고려하여 분량을 조절해야 합니다.
- **상대 경로**: 원고 내 `<img>` 태그 삽입 시 반드시 상위 폴더를 참조하는 상대 경로(`../../images/...`)를 사용하여 네이버 블로그 업로드 시 호환성을 유지합니다.
- **일괄 처리**: 위 명령 한 번으로 폴더 내의 모든 원고(보통 5~10개)를 한꺼번에 처리할 수 있습니다.
