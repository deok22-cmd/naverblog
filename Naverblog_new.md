## 02 | 주제 선정 및 트래픽 전략 (2026-04-22 최적화)

### 2.1. 콘텐츠 데일리 믹스 전략 (4+1 시스템)
블로그의 안정적인 유입과 전문성을 동시에 확보하기 위해 하루 생산 원고를 총 **5개**로 제한하고 아래 비중을 강제한다.
- **[Main] 국내 여행, 축제, 추천 관광지 (80%)**: 대량의 실시간 유입 및 계절감 확보 (매일 4개)
- **[SUB] 기술 권위 & 라이프스타일 (20%)**: 전문가적 이미지 구축 (매일 1개)
  - **스프레드시트 강의**와 **요리 레시피**를 매일 번갈아 가며 진행 (격일제 로테이션)

### 2.2. 초정밀 팩트체크 원칙 (Absolute Accuracy)
모든 분야는 신뢰성이 생명이다. 다음 사항을 반드시 `search_web`으로 검증 후 기재한다.
1.  **수치 데이터**: 자재 사양, 가격, 법령상의 면적/거리, 운영 시간 등.
2.  **고유 명사**: 현지 지명(원어 병기), 모델명, 식물 학명 등.
3.  **논리적 근거**: "카더라"가 아닌 공식 제조사 매뉴얼이나 정부 공고문 기반 서술.

**※ 주의사항: 모든 원고 작성 전 반드시 `1만명_달성_액션플랜.md`를 상호 참조하여 전략적 정렬을 확인한다.**

### 2.3. 서브 주제 로테이션 원칙
- **순서 관리**: `sub_topic_tracker.md` 파일을 참조하여 오늘이 스프레드시트 차례인지 레시피 차례인지 확인한다.
- **진행 방식**: 
  - 스프레드시트: `spreadsheet.md` 리스트 중 `⏳ 대기` 상태인 최상단 주제 선정.

- **작업 완료 후**: 해당 주제 파일(`spreadsheet.md` 또는 `receipt.md`)의 상태를 `✅ 완료`로 변경하고, `sub_topic_tracker.md`에 오늘의 발행 이력을 기록한 뒤 다음 차례 항목을 업데이트한다.

## 04 | [플래티넘 스탠다드] 포맷 & 디자인 v5 (Editor-Safe)

### 핵심 원칙
**네이버 블로그 에디터 복사-붙여넣기 시 디자인 무결성을 유지하며, 압도적인 전문성을 확보한다.**

1. **에디터 세이프 디자인 (v5)**: 
   - `grid`, `flex` 등 복잡한 CSS 금지. `<table>`과 단순 스타일만 사용.
   - **[핵심]** 모든 텍스트 하단에 밑줄이 생기지 않도록 `border-bottom` 사용을 자제하고, 하이퍼링크는 `text-decoration: none;`을 강제한다.
2. **고밀도 내러티브 & 정보 집약 (꼭지 9+ 규칙)**: 
    - 소제목(h2, h3) **9개 이상** 및 각 꼭지별 **15~20행** 서술.
    - 문단 사이 **`<br><br>`** 삽입으로 가독성 확보.
3. **내부 링크 (T-1 낙수전략)**: 어제 발행된 원고 2개 랜덤 링크.
4. **강조 및 지침**: `**` 금지, **`<strong>`** 태그만 사용.

### [표준 템플릿 v5.3 - 라이트]
- **주의**: 에디터 복사 시 `<html>`, `<head>`, `<body>` 태그를 제외하고 내부 `<div>`부터 복사하여 붙여넣는다.

```html

  <h1 style="font-size: 1.6em; font-weight: 800; color: [컬러-다크]; text-align: center; margin-bottom: 30px;">제목</h1>

    <strong>리드문</strong>

  <h2 style="font-size: 1.2em; font-weight: 700; background: [컬러-연하게]; padding: 12px 15px; border-left: 6px solid [컬러]; margin-top: 50px; color: [컬러-다크];">1. 소제목</h2>
```

## 06 | Z.E.P 1.0 프로토콜 (Zero Error Protocol)

### 6.1. 카테고리별 필수 검증 앵커 (Anchor points)
*   **[국내 여행 & 축제]**: 실시간성(2026년 공지 확인), 정보 정합성(개장일, 입장료, 주차 현황 등) 교차 검증.
*   **[IT/테크/실무]**: 스펙 정확도(Hallucination 금지), 실존 인격 확보(실존 모델/사양서 기반).
*   **[맛집/레시피]**: 재료의 화학적 변화(마야르 반응 등)와 조리 도구의 온도를 수치로 상세 설명.

### 6.2. 3대 원고 생성 금지 원칙 (Forbidden Rules)
1. **Hallucination 수치 금지**: 구체적 수치는 반드시 공식 사양서 검색 후 기재.
2. **절대값 확신 금지**: 변수가 많은 현상에 대한 절대적 단정 금지 (대신: "높은 확률로" 등 사용).
3. **경쟁사 평가절하 금지**: 비교 시 타 업체를 일방적으로 폄하하지 않음.

### 6.3. 원고 생성 전 '자가 점검' 리스트
- "이 수치/날짜는 2026년 공식 데이터인가?"
- "이 내용은 제조사/정부기관의 오피셜 공지인가?"
- "독자가 내 글을 보고 행동했을 때 리스크가 없는가?"

   - **[핵심]**: 상태 변경과 동시에 대시보드 리스트의 **기사 제목**에도 해당 원고 파일(HTML)로의 하이퍼링크를 반드시 연결한다.

### 8.2. 이미지 생성 및 관리 규칙 (v3.8 업데이트)
1. **저장 경로**: 모든 이미지는 `d:\lightsail\naverblog\images\yymmdd\`에 저장한다.
2. **원고 내 참조**: 반드시 상대 경로(`../../images/yymmdd/파일명.png`)를 사용한다.
3. **네이밍**: `원고seq_keyword_imageseq.png` 규칙 준수.
4. **출력 형태 (HTML)**:
   - **실제 이미지 삽입 시**: `<table>` 등을 사용하지 않고 `<img>` 태그를 독립적으로 상단에 배치한다.
   - **Placeholder(이미지 미생성) 시**: 디자인 정돈을 위해 `<table>` 내부에 영문 프롬프트와 복사 버튼을 배치한다.
   - **캡션 공통**: 이미지 바로 아래에 한 줄 설명과 **"AI 제작 이미지"** 문구를 포함한 캡션을 텍스트로 출력한다.
5. **이미지 자동 생성**: 쿼터 허용 시 원고당 3장 생성. 캡션은 `[키워드] / AI 제작 이미지` 필수 표기.
- **[절대 금지]**: 본문 내 운영자용 메타 정보(쿼터 초과 등) 노출 금지.
6. **프롬프트 복사 기능 (Placeholder)**: 쿼터 소진 시 복사 버튼과 다음 스크립트 필수 포함.
```javascript
function copyPrompt(id) {
  const text = document.getElementById(id).innerText;
  navigator.clipboard.writeText(text).then(() => {
    const btn = document.querySelector(`button[onclick="copyPrompt('${id}')"]`);
    const originalText = btn.innerText;
    btn.innerText = "복사 완료!";
    btn.style.background = "#27ae60";
    setTimeout(() => { btn.innerText = originalText; btn.style.background = "#00796b"; }, 2000);
  });
}
```
